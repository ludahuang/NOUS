# Memory Policy

## Memory Classes

- **Source memory**: registered historical or public material with provenance.
- **Project memory**: decisions and artifacts for a specific active project.
- **Personal memory**: information about an individual; private by default.
- **Ephemeral context**: temporary working material that should not persist.

## Rules

- do not convert ephemeral or personal context into durable memory without a
  clear reason and permission;
- keep source URLs, capture dates, and rights with durable source memory;
- allow project owners to inspect and remove project memory;
- do not train or publish from private memory by default;
- record when generated summaries replace or supersede earlier summaries;
- preserve original source records when interpretation changes.
