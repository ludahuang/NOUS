# Projects

Projects are runnable, deployable, or independently publishable NOUS works.

Each project should contain:

- `PROJECT.md` for intent and boundaries;
- a catalog entry under `catalog/projects/`;
- source, runtime, and preservation information;
- project-specific tests and release instructions;
- an explicit relationship to NOUS protocols and agents.

The first project boundary is [`the-vault/`](the-vault/).
