# Authority Policy

## Allowed by Default

- read explicitly registered public sources;
- compare, classify, and summarize with citations;
- suggest relationships and missing records;
- draft research notes, manifests, project plans, and creative directions;
- prepare code changes inside an approved workspace;
- run reversible validation and local tests.

## Requires Explicit Approval

- external publication or deployment;
- deletion or replacement of source material;
- changing permissions, ownership, or licenses;
- importing private material into a persistent memory;
- spending money or creating paid resources;
- merging or rewriting historical Git branches;
- representing generated content as an original historical record.

## Prohibited

- hiding provenance;
- inventing rights or permissions;
- silently collapsing conflicting interpretations;
- using private sources outside their approved purpose;
- removing a preservation record solely because a newer version exists.
