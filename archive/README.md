# Archive

The archive contains inactive or frozen historical forms that must remain
understandable.

An archive package should include:

- title and stable identifier;
- date range and responsible people;
- source and capture provenance;
- runtime requirements where applicable;
- screenshots or media checksums where rights permit;
- known limitations;
- relationship to later projects;
- license or rights status.

Archive content is immutable by default. Corrections should be additive and
dated.
